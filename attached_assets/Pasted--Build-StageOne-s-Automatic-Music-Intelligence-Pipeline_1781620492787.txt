# Build StageOne’s Automatic Music Intelligence Pipeline

Do not use n8n.

Do not add unrelated features.

Use the available partner APIs directly from the StageOne backend:

* Musixmatch
* ElevenLabs
* LALAL.AI
* Cyanite

The goal is to transform the current manual upload flow into an intelligent creator experience:

**Upload performance → isolate vocals when needed → transcribe the singing → identify the song → detect the lyrics actually performed → synchronize the lyrics with the video → analyze mood and genre → let the creator confirm and publish.**

This must work from the creator’s actual uploaded video. Do not use hardcoded Adele data or seeded analysis results.

---

# 0. Resolve the current publishing blocker first

The current physical-device flow still ends with:

`Couldn’t verify your upload. Please try again.`

Before calling the intelligent pipeline complete, fix the upload confirmation and publishing path.

The required publishing path is:

1. Request storage upload session
2. Upload the video
3. Verify the stored object
4. Create the post database record
5. Save analysis metadata
6. Show the post in Home
7. Show it in Profile
8. Play it after app restart

The intelligence pipeline may analyze the stabilized local video before final publication, but the full result is not complete until the post publishes successfully.

# 4. Use LALAL.AI for vocal isolation

Use LALAL.AI when the uploaded performance contains:

* Loud backing music
* Instrumental accompaniment
* Background noise
* Another recorded singer
* Audio that produces low transcription confidence

Preferred behavior:

1. Extract audio from the uploaded video
2. Send the audio to LALAL.AI
3. Request vocal isolation or vocal/instrumental separation
4. Retrieve the vocal stem
5. Use the vocal stem for transcription
6. Delete temporary stem files when analysis finishes

Do not always run LALAL.AI when it is unnecessary.

Use this decision strategy:

* Clear a cappella or clean vocal → transcribe directly
* Music-backed performance → isolate vocals first
* Direct transcription has low confidence → retry using isolated vocals
* LALAL.AI unavailable → continue with the original audio and report lower confidence

Store only:

* Whether vocal isolation was used
* Separation status
* High-level quality/confidence result

Do not permanently store the isolated vocal by default.

# 5. Use ElevenLabs for singing transcription

Use ElevenLabs speech-to-text capability to transcribe the original or isolated vocal audio.

Request word-level timestamps if supported by the current API access.

Normalize the response into:

* Full recognized text
* Detected language
* Word text
* Word start time in the uploaded video
* Word end time
* Confidence when available
* Phrase or segment timestamps

Handle singing-specific behavior:

* Extended vowels
* Repeated words
* Missing words
* Accent changes
* Different tempo
* Background instruments
* Partial lyrics
* Silence before singing
* Spoken introductions
* Ad-libs

Do not expect a perfect transcript.

Filter low-confidence filler words carefully, but do not remove meaningful lyric fragments.

Do not permanently store the complete transcript unless required.

Persist only compact recognition and alignment metadata needed for the post.

---

# 6. Use Musixmatch to identify the song

Use the ElevenLabs transcript to search Musixmatch.

Do not expect Musixmatch alone to identify raw video audio unless a documented audio-recognition capability is actually available to this project.

Create several distinctive transcript phrases.

Avoid searching only with common words such as:

* Love
* Baby
* Tonight
* Yeah

Prefer longer distinctive lyric sequences.

Search Musixmatch using available lyric-search capabilities such as:

* Lyric-text search
* General track search
* Language
* Artist or title hints when the creator entered them
* Tracks that have lyrics
* Popularity or confidence ranking

For each candidate:

1. Fetch official track metadata
2. Fetch RichSync when available
3. Fall back to subtitles
4. Fall back to plain lyrics
5. Compare the official lyrics with the ElevenLabs transcript
6. Score consecutive phrase matches
7. Penalize weak matches containing only common words
8. Detect repeated chorus occurrences
9. Produce a confidence score

Use a real ranking algorithm rather than selecting the first result.

# 10. Align the lyrics to the uploaded performance

The uploaded cover may not use the original track’s exact tempo.

Do not rely only on one global timing offset.

Use multiple alignment anchors.

For every confidently matched word or phrase, record:

* Uploaded-video timestamp from ElevenLabs
* Official lyric timestamp from Musixmatch RichSync or subtitles
* Match confidence

Preferred timing modes:

1. `richsync_aligned`

   * Multiple word-level anchors

2. `subtitle_aligned`

   * Multiple line-level anchors

3. `linear_scaled`

   * Tempo scale plus offset

4. `offset_only`

   * One global offset when limited anchors exist

5. `plain`

   * Lyrics detected but no reliable automatic timing

Use piecewise alignment when tempo changes or pauses occur.

Do not fabricate word synchronization when only plain lyrics exist.
