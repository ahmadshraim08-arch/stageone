# StageOne — Real Multi-User Testing Environment

I need StageOne to support real testing across multiple physical phones.

Do not treat this as a single-device demo anymore.

I want to create real accounts on my phone, my brother’s phone, and my sister’s phone so we can test the app as separate users at the same time.

The purpose is to verify that StageOne behaves like a real social network with shared backend data, persistent accounts, following, direct sharing, messaging, uploads, Golden Mics, and Musixmatch-powered features.

Before coding, inspect the current authentication, backend, database, storage, API routes, and app state.

Do not create another isolated mock-data system if a real backend can be used.

---

# Main Goal

Create a stable test environment where at least three real users can:

1. Create separate accounts from different phones.
2. Log in and log out independently.
3. Create and edit profiles.
4. Follow one another.
5. See follower and following updates on both sides.
6. Upload Music Minutes.
7. See each other’s Music Minutes in the feed and on profiles.
8. Like, comment, save, and give Golden Mics.
9. Share Music Minutes directly with one another.
10. Send and receive text messages.
11. See unread message indicators.
12. Test Musixmatch search, lyrics, sections, translations, mood, and “Sing This Part.”
13. Confirm all data remains after closing and reopening the app.

The app must use one shared backend and database so actions performed on one phone appear on the other phones.

---

Implement or stabilize real authentication.

Use the project’s existing authentication provider if one already exists.

If authentication is currently mocked or stored only in local state, replace it with a real backend authentication system that works across devices.

Acceptable options include:

* Supabase Auth
* Firebase Auth
* Clerk
* Replit-compatible email/password authentication
* The existing backend auth system if already implemented correctly

Required account features:

* Sign up with email and password
* Log in with email and password
* Log out
* Persistent session
* Unique user ID
* Unique username
* Duplicate-email protection
* Duplicate-username protection
* Password validation
* Clear authentication errors
* Session restoration after app restart

Optional if easy:

* Email verification
* Password reset

Do not use one shared demo user for all phones.

# 2. Shared Test Environment

The backend must be reachable from multiple physical phones.

Do not use:

```text
localhost
127.0.0.1
```

as the mobile API address.

Use a deployed or externally reachable Replit backend URL.

Create a clear environment configuration such as:

```env
EXPO_PUBLIC_API_URL=https://your-stageone-api.replit.app
MUSIXMATCH_API_KEY=server-side-only
```

The Musixmatch key must remain on the backend.

The mobile app should use the public API base URL.

Required:

* HTTPS endpoint
* Shared database
* Shared file/video storage
* No device-specific local database as the source of truth
* Clear handling when the backend is offline

Before completion, verify that three separate phones can connect to the same backend.

# 4. Following and Mutual Connections

Implement real follow relationships between accounts.

Required behavior:

* User A can follow User B.
* User B receives a notification.
* User A’s following count increases.
* User B’s follower count increases.
* The relationship remains after restarting both apps.
* User A can unfollow User B.
* Counts update correctly.
* Following feed shows posts from followed users.

Suggested database model:

```ts
type Follow = {
  id: string;
  followerId: string;
  followingId: string;
  createdAt: string;
};
```

Prevent duplicate follow records.

Prevent a user from following themselves.

If User A follows User B and User B follows User A, label the relationship as:

**Friends**

This mutual-follow state can be used in the share and messaging UI.

```ts

  participantIds: string[];
  lastMessageAt: string;
  createdAt: string;
};
```

```ts
type Message = {
  id: string;
  conversationId: string;
  senderId: string;
  type: "text" | "music_minute_share";
  text?: string;
  musicMinuteId?: string;
  createdAt: string;
  readAt?: string;
};
```

Prevent duplicate one-to-one conversations between the same two users.
