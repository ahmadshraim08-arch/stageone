3. Use ElevenLabs direct video transcription as the primary path

The ElevenLabs Speech-to-Text endpoint supports major audio and video formats.

Change the primary pipeline to:

1. Obtain the uploaded video’s readable file, stream, or temporary signed read URL.
2. Send that video directly to ElevenLabs Speech-to-Text.
3. Request word-level timestamps.
4. Allow automatic language detection.
5. Parse the transcript.
6. Continue to Musixmatch matching.

Do not require FFmpeg before ElevenLabs unless direct transcription fails for a confirmed format reason.

Preferred input order:

1. Server-readable App Storage source_url
2. Stream or multipart upload from the server
3. Temporary local video file
4. FFmpeg-extracted WAV only as a fallback

Do not send the device-local file:// URI to the backend or ElevenLabs.

Do not convert the complete video to Base64.

7. Validate the transcript before Musixmatch matching

A successful ElevenLabs response is usable only when:

* Transcript text is non-empty
* A reasonable number of words exists
* Word timestamps are valid
* The output is not only audio-event labels
* Detected singing words contain enough usable lyric content

Filter:

* Spacing entries
* Empty entries
* Non-speech event labels
* Repeated filler
* Very low-value single words

Preserve real repeated lyric phrases when meaningful.

If the transcript is too short, display:

We heard the performance, but there weren’t enough clear words to identify the song. Search manually.

Do not call Musixmatch with an empty transcript.

8. Build reliable Musixmatch lyric queries

Use the ElevenLabs transcript to generate several distinctive search phrases.

Do not submit the entire noisy transcript as one query.

Generate candidates from:

* Consecutive groups of meaningful words
* High-confidence words
* Uncommon lyric phrases
* Phrases from different points in the performance

For each phrase, call Musixmatch track search using:

* q_lyrics
* f_has_lyrics=1
* Language filter when helpful
* Popularity sorting when helpful

Combine and deduplicate candidate tracks.

For each candidate:

1. Fetch track metadata.
2. Fetch RichSync.
3. Fall back to subtitles.
4. Fall back to plain lyrics.
5. Compare the official lyrics against the complete transcript.
6. Score consecutive phrase overlap.
7. Score total word overlap.
8. Penalize common-word-only matches.
9. Detect repeated chorus occurrences.
10. Return a confidence score.

Do not automatically select a weak result.
