You are Replit Agent. Build a full-stack MVP web app called **StageOne**.

StageOne is a TikTok-style music discovery platform built specifically for aspiring singers and musicians. The core idea is:

**Early TikTok became viral through dancing. StageOne is built to go viral through singing.**

StageOne should feel like a real social media app, not a formal audition website. Anyone can open the app, scroll short vertical singing videos, like, comment, share, follow, and support singers. Anyone can also choose to participate by posting their own short singing video called a **Music Minute**.

The goal is to make StageOne the go-to app for people who want to become singers, musicians, or performers. The experience should feel addictive, authentic, mobile-first, social, and music-native.

The app should visually match the uploaded UI/UX concept images:

* Dark premium concert-style theme
* Black and deep purple backgrounds
* Neon purple and pink accents
* Gold accents for Golden Mics
* Modern TikTok-style mobile interface
* Glowing microphone iconography
* Stage lights, music waveforms, and live performance energy
* Clean rounded cards, bold typography, and high-contrast UI
* A polished product demo feel suitable for Musicathon submission

Do not create a generic app. Build StageOne as a premium social music product.

---

# 1. Product Positioning

StageOne is a fan-powered singing discovery app where emerging singers upload short vertical performance videos, get discovered in a TikTok-style feed, and earn support from fans through Golden Mics.

One-liner:

**StageOne lets emerging singers upload short Music Minutes, get discovered in a TikTok-style feed, and earn Golden Mics from fans to rise toward live competition.**

Longer explanation:

StageOne gives every aspiring singer their first digital stage. Instead of waiting for labels, managers, TV talent shows, or random viral luck, singers can upload 60-second performances, build traction directly from fans, and climb through the platform based on engagement and support.

The app should feel familiar like TikTok, but focused only on singing, music performance, lyric-powered creation, and emerging talent discovery.

# 3. Main User Experience

The app should mimic the simplicity of TikTok.

Users should not be forced into complicated roles at signup. Everyone starts as a normal user. A user can watch, like, comment, share, follow, and support. If they want to create, they tap **Post** and upload a Music Minute.

Main user loop:

**Open app → scroll Music Minutes → discover singer → like/comment/share/follow → give Golden Mic → post your own Music Minute → climb Rise Chart → unlock StageOne Live eligibility**

The product must feel fun even if a user never posts. Watching should be enough. Posting should be optional but easy.

## 5.5 Musixmatch Integration

Musixmatch must be meaningful and visible because this is a Musicathon project.

Build Musixmatch as the music intelligence layer of StageOne.

Use the API server-side only. Do not expose the API key in the frontend.

Environment variable:
MUSIXMATCH_API_KEY=

Backend routes:

* GET /api/musixmatch/search?q=
* GET /api/musixmatch/track/:id
* GET /api/musixmatch/lyrics/:id
* GET /api/musixmatch/richsync/:id
* GET /api/musixmatch/translations/:id if available

Use Musixmatch for:

* Track search during upload
* Track matching for covers
* Song metadata cards
* Lyric context
* Synced lyric display if available
* Lyric translation if available
* Discover by song
* Challenge ideas based on song/lyrics

Important:
Do not permanently store full lyrics from Musixmatch.
Do not cache or bulk download lyrics.
Store only minimal reference metadata:

* Musixmatch track ID
* Track title
* Artist name
* Album name if needed
* Language if needed
* Lookup timestamp

Display disclaimer wherever lyrics appear:
**Lyrics and metadata powered by Musixmatch. Demo use only.**

Create a feature called **LyricStage**:
LyricStage lets creators search for a song, attach official song metadata, and display lyric context or synced lyrics during recording/playback.

## 5.6 Golden Mics

Golden Mics are the core fan support mechanic.

They are not normal likes.

A like means:
“I enjoyed this.”

A Golden Mic means:
**“I believe this singer deserves to rise.”**

Rules:

* Every user starts with 1 free Golden Mic.
* User can give a Golden Mic to a Music Minute.
* Giving a Golden Mic increases the creator’s Golden Mic count.
* Golden Mics affect Rise Score and Rise Chart ranking.
* If the user has no Golden Mics left, show a modal to get more.

For the Musicathon MVP, purchases are simulated only. No real payments.

The modal should say:
**Musicathon demo: Golden Mic purchases are simulated. No real money is collected.**

Simulated packages:

* 1 Golden Mic — $0.99 demo
* 5 Golden Mics — $3.99 demo
* 15 Golden Mics — $9.99 demo
* 50 Golden Mics — $24.99 demo
* 100 Golden Mics — $39.99 demo

When a user gives a Golden Mic, show a toast:
**Golden Mic sent. You helped this artist rise.**

Golden Mic button should look special:

* Gold microphone icon
* Glow effect
* Small animation/toast on click

## 5.7 Rise Chart

Rise Score =
(Golden Mics * 10) +

(Likes * 1) +
(Comments * 2) +
